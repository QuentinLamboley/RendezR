# Roadmap

## v0.3.0 candidates

- optional mDNS / multicast discovery backend where an audited cross-platform implementation is available;
- explicit pairing approval prompt rather than immediate acceptance by an available peer;
- optional room passphrase with a memory-hard verifier;
- encrypted transport for controlled, certificate-managed local deployments;
- RStudio addins for start, random match, send and stop;
- accessibility and localized console messages.

## Non-goals

- a public global random-chat network;
- server-side accounts or behavioural analytics;
- sending R code for automatic execution;
- sending files, screen captures or sensitive data;
- bypassing client isolation, firewall policies or network administrator controls.
